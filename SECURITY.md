# Security

Не публикуйте токены Telegram, пароли PostgreSQL, `.env`, резервные копии или данные покупателей в issues и pull requests.

Для сообщения об уязвимости используйте приватный GitHub Security Advisory репозитория. Укажите затронутую версию, шаги воспроизведения и возможное влияние. Не проверяйте уязвимость на реальных данных покупателей без разрешения владельца системы.
